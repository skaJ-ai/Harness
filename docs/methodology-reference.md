## 1. 문서 구조화 방법론

### 피라미드 원칙 (Pyramid Principle)
- **출처**: Barbara Minto (McKinsey, 1973)
- **구조**: 결론 먼저(Top-down) → 하위 근거 3~5개 MECE 그룹핑 → 수직(Why?/So What?) + 수평(연역/귀납)
- **적합**: 경영진 보고, 의사결정 요청 문서
- **산파술**: "결론부터 말해주세요. 이 보고서로 상대방이 어떤 행동을 하길 원하세요?"

### SCQA (Situation-Complication-Question-Answer)
- **출처**: Barbara Minto, McKinsey
- **구조**: S(현재 상황) → C(문제/변화) → Q(핵심 질문) → A(답=보고서 전체)
- **적합**: 보고서 도입부, Executive Summary, 발표 오프닝
- **산파술**: "지금 상황은?" → "문제가 뭔가요?" → "이 보고서가 답할 질문은?" → "당신의 답은?"

### PREP (Point-Reason-Example-Point)
- **출처**: 비즈니스 커뮤니케이션 표준
- **구조**: 주장 → 이유 → 사례/데이터 → 주장 재확인
- **적합**: 보고서 개별 섹션, 이메일, 구두 보고
- **산파술**: 근거 없는 주장 시 "왜 그렇게 생각하세요?" / 추상적일 때 "구체적 사례나 숫자가 있나요?"

### SDS (Summary-Detail-Summary)
- **구조**: 요약 → 상세 → 요약 반복
- **적합**: 사내 공지, 짧은 보고서, 교육 자료
- **산파술**: "먼저 한 줄 요약부터 만들고, 상세를 채우고, 마지막에 다시 정리하는 구조로 갈까요?"

## 2. 문제 분석 / 현황 파악

### SWOT
- **출처**: Albert Humphrey (1960~70년대)
- **구조**: Strengths / Weaknesses / Opportunities / Threats + TOWS 매트릭스
- **산파술**: "내부 강점은?" → "약점은?" → "외부 기회는?" → "위협은?" → "강점으로 기회를 잡으려면?"

### As-Is / To-Be
- **출처**: BPR, 컨설팅 표준
- **구조**: 현재 상태 → Gap 식별 → 목표 상태 → Action Plan
- **산파술**: "지금 상태를 설명해주세요" → "이상적으로는?" → "가장 큰 차이는?" → "메우려면?"

### 5 Whys
- **출처**: 도요타 (Sakichi Toyoda, 1930년대)
- **구조**: "왜?" 5회 반복 → 근본 원인(Root Cause) 도달
- **산파술**: "왜 그런 일이 발생했을까요?" 반복. 표면 → 근본으로 파고들기

### Fishbone (Ishikawa)
- **출처**: Kaoru Ishikawa (1968)
- **구조**: 중심=문제, 가지=원인 영역(6M: Man/Machine/Method/Material/Measurement/Environment)
- **산파술**: "이 문제의 원인을 인력, 프로세스, 시스템, 환경으로 나눠볼까요?"

### PEST/PESTLE
- **출처**: Francis Aguilar (1967)
- **구조**: Political / Economic / Social / Technological (+ Legal / Environmental)
- **산파술**: 외부 환경 분석 시 "정치/규제, 경제, 사회, 기술 측면에서 각각 어떤 변화가 있나요?"

### Force Field Analysis
- **출처**: Kurt Lewin (1940년대)
- **구조**: 추진력(Driving) vs 저항력(Restraining) → 전략 도출
- **산파술**: "이 변화를 밀어줄 힘은?" / "반대하거나 방해하는 요인은?"

### 3C
- **출처**: Kenichi Ohmae (1982)
- **구조**: Company / Customer / Competitor
- **산파술**: "우리 회사, 고객, 경쟁사 세 관점에서 각각 정리해볼까요?"

## 3. 기획 / 제안

### MECE
- **출처**: McKinsey, Barbara Minto
- **구조**: 상호 배타(겹침 없음) + 전체 포괄(빠짐 없음)
- **산파술**: "이 분류에서 겹치는 건 없나요? 빠진 건 없나요?" — 모든 구조화의 검증 도구

### Issue Tree
- **출처**: 컨설팅 표준
- **구조**: 핵심 질문 → MECE 하위 분해 → 분석 가능 단위까지
- **산파술**: "이 문제를 큰 덩어리로 나눠볼까요? 크게 어떤 영역으로 나뉘나요?"

### Design Thinking (5단계)
- **출처**: IDEO / Stanford d.school
- **구조**: Empathize → Define → Ideate → Prototype → Test
- **산파술**: "이 제도를 쓰는 사람 입장에서 가장 불편한 게 뭘까요?" (Empathize)

### OKR
- **출처**: Andy Grove → John Doerr → Google
- **구조**: Objective (질적 목표) + Key Results (정량 지표 3~5개)
- **산파술**: "궁극적으로 달성하고 싶은 건?" → "달성 여부를 숫자로 표현하면?"

### Logic Model
- **출처**: W.K. Kellogg Foundation (2004)
- **구조**: Input → Activity → Output → Outcome → Impact
- **산파술**: "투입 자원은?" → "실행할 활동은?" → "직접 결과물은?" → "궁극적으로 뭐가 변하나요?"

### 6W2H
- **출처**: 5W1H 확장
- **구조**: What / Why / Who / Whom / When / Where / How / How much
- **산파술**: 기획 초기에 8가지 질문으로 기본 골격 완성

## 4. 컨설팅 보고서 기법

### Hypothesis-Driven
- **출처**: McKinsey
- **구조**: 초기 가설 → Issue Tree 분해 → 데이터로 검증 → 수정 → 권고안
- **산파술**: "직감적으로 원인이 뭐라고 생각하세요?" → "그게 맞다면 어떤 증거가 있어야 할까요?"

### So What / Why So
- **출처**: McKinsey
- **구조**: So What?(위로: 의미 도출) / Why So?(아래로: 근거 확인)
- **산파술**: 데이터 제시 시 "이게 의미하는 바는?" / 주장 시 "어떤 근거로?"

### Storyline
- **출처**: McKinsey, BCG
- **구조**: 각 섹션 핵심 메시지(headline) → 순서대로 읽으면 하나의 이야기
- **산파술**: "각 섹션의 핵심을 한 문장씩 적어볼까요? 이어서 읽었을 때 자연스러운가요?"

### Action Title
- **출처**: 컨설팅 공통
- **구조**: 섹션 제목을 서술문으로 (Bad: "매출 현황" → Good: "온라인 매출이 전년 대비 23% 성장")
- **산파술**: "이 섹션에서 말하고 싶은 핵심을 문장으로 바꿔볼까요?"

## 5. HR 특화

### Kirkpatrick 4단계 ⭐ (교육 평가)
- **출처**: Donald Kirkpatrick (1959)
- **구조**: Level 1 반응(만족도) → Level 2 학습(지식/스킬) → Level 3 행동(현업 적용) → Level 4 결과(사업 성과)
- **산파술**: "이 교육의 효과를 어떻게 측정할 건가요? 만족도만 볼 건가요, 실제 행동 변화까지?"
- **HARP 적용**: "교육 운영 결과 요약" 템플릿의 핵심 구조로 직접 사용

### 역량 모델링
- **출처**: David McClelland (1973), Spencer & Spencer (1993)
- **구조**: 핵심 역량 / 리더십 역량 / 직무 역량 → 각각 정의 + 행동 지표 + 수준
- **산파술**: "이 직무에서 가장 중요한 역량 3~5가지는?"

### Employee Journey Mapping
- **출처**: Customer Journey Mapping의 HR 적용 (2010년대)
- **구조**: Attract → Recruit → Onboard → Develop → Perform → Reward → Retain → Exit
- **산파술**: "직원 입장에서 가장 불만이 큰 단계는 어디인가요?"

### 9-Box Grid
- **출처**: McKinsey-GE (1970년대)
- **구조**: X=성과 / Y=잠재력 → 9칸 인재 분류
- **산파술**: "성과와 잠재력 두 축으로 인재를 분류해볼까요?"

### HR Analytics 4단계
- **출처**: Jac Fitz-enz (2000)
- **구조**: Descriptive(현황) → Diagnostic(원인) → Predictive(예측) → Prescriptive(처방)
- **산파술**: "현황 보고인가요, 원인 분석인가요, 예측/제안인가요?"
